What is This Project About?

Feelsify.io is a web that allows users to record their daily levels of anxiety, stress, and depression. Users will also be able to include brief notes that might be relevant to how they are feeling.


The app will store this information and provide users with easy-to-read graphs that will display trends in their mood.


This app is ideal for users who want to stay on top of their mental health and be able to recognize any prolonged, negative trends of anxiety, stress, or depression, which can be detrimental to their health and well-being.


In the future, we will add quizzes that hope to help users determine whether or not they might be experiencing anxiety, stress, depression, and even burn out.

RUNNING THE PROJECT LOCALLY

In order to run the project locally, both client and server app needs to be running.

Starting the Client side

  • navigate to ./client
  • Run npm install to install all dependencies
  • Run npm start to start in development mode (default port is 3000 on localhost)

Starting the Server side

  • Navigate to ./server
  • Run npm install to install all dependencies first

    • From here, You can start the server in 2 ways:

    1. clear && PORT=3001 node index.js

    This command starts the server on port 3001. Open http://localhost:3001/api to view it in your browser. Since React client runs on port 3000 by default, it is recommended to specify the port number when you start the server.

    2. Start the server with ./server/start_server.sh

    Run ./server/start_server.sh in the terminal. This will automatically start the server on port 3001. You may first need to make the bash script executable with the command chmod u+x start_server.sh

    How to login

    You can use any account in ./client/mockData.js to login. Just use the values of username and password. The easiest account to login is: username: test123 password: test

    NOTE

    Server does not work with the browser The server has been designed to work with the client, not with the browser. You can use the program "POSTMAN" to function as an alternative client for development and testing purposes. To see all the available routes and API calls for the server go to APIrouter.js (routes) and mockAPI.js (functions).

    PS: As of now, there is no official database yet. All non hard-coded user data will be lost when the server restarts.
